# Haven 🌱

> Growing through the journey, together.

A safe place for students to share struggles, celebrate wins, and realize they are not alone.

Built with TanStack Start, React 19, Tailwind v4, Lovable Cloud (Supabase), and Capacitor (Android).

## Android APK — Full Build Flow

### Architecture

Haven uses TanStack Start's **server functions** (`createServerFn`) for the backend.
Those run on Lovable's edge runtime, not inside the APK. So the Android build:

1. **Bundles the web UI** locally inside the APK (loads instantly, no white screen).
2. **Calls the remote Lovable backend** for all auth / data / server-function requests.

The remote backend URL is baked into the build via `VITE_REMOTE_API_BASE`
(defaults to your project's published Lovable URL). Native fetch is patched
in `src/lib/native-fetch-bridge.ts` so every relative request is rewritten
to `${VITE_REMOTE_API_BASE}${path}` when running inside the APK.

### Prerequisites (one-time, on your machine)

- Node 20+ and `bun` (or npm)
- **Android Studio** (latest stable) with:
  - Android SDK Platform 34+
  - Android SDK Build-Tools
  - An emulator or a physical device with USB debugging enabled
- **JDK 17** (ships inside Android Studio)

> The Lovable sandbox cannot run the Android SDK / Gradle, so the APK must
> be built on your own machine. The `android/` folder and the web build are
> generated by the commands below.

### Step-by-step

```bash
# 1. Clone & install
git clone <your-repo-url> haven
cd haven
bun install

# 2. (Optional) point the APK at a different backend
echo 'VITE_REMOTE_API_BASE=https://project--47f00b74-0ab7-433f-a843-5ac5b0f9b92c.lovable.app' >> .env

# 3. Build the web bundle + Capacitor index.html
bun run cap:build
#   → dist/client/   (what Capacitor packages into the APK)

# 4. Add the native Android project (only the very first time)
npx cap add android

# 5. Copy the web build into the native project
npx cap sync android
#   (shortcut: bun run cap:sync  — does steps 3 + 5 together)
```

### Generate the debug APK

**Option A — Android Studio (recommended):**

```bash
bun run cap:open       # or: npx cap open android
```

In Android Studio:
1. Wait for Gradle sync to finish.
2. `Build` → `Build Bundle(s) / APK(s)` → `Build APK(s)`.
3. Click **locate** in the popup. APK is at
   `android/app/build/outputs/apk/debug/app-debug.apk`.

**Option B — Command line:**

```bash
cd android
./gradlew assembleDebug
# APK: android/app/build/outputs/apk/debug/app-debug.apk
```

Install on a connected device:

```bash
adb install -r android/app/build/outputs/apk/debug/app-debug.apk
```

### Signed release APK

```bash
cd android
./gradlew assembleRelease
```

You'll need a keystore — see
[Capacitor's signing guide](https://capacitorjs.com/docs/android/deploying-to-google-play#signing-an-apk).

### Updating after web changes

Every time you change the web app:

```bash
bun run cap:sync
# then rebuild the APK from Android Studio (or ./gradlew assembleDebug)
```

### App icon & splash screen

The default Capacitor icon and a splash with background `#F5F1E8` (Haven cream)
are configured. To replace them with custom artwork:

1. `bun add -D @capacitor/assets`
2. Drop a 1024×1024 `icon.png` and a 2732×2732 `splash.png` into a top-level
   `assets/` folder.
3. `npx @capacitor/assets generate --android`
4. `bun run cap:sync`

### Google Sign-In on Android

Google OAuth works out-of-the-box in the **web** build. For the **APK** you
must add your debug keystore's SHA-1 fingerprint to the Google OAuth client:

```bash
keytool -list -v -keystore ~/.android/debug.keystore \
  -alias androiddebugkey -storepass android -keypass android
```

Add the printed `SHA1` to your Google Cloud OAuth client → "Authorized
Android apps" with package name `com.haven.app`. Email/password sign-in
works without any extra setup.

## What was installed / configured

**npm packages (already in `package.json`):**

- `@capacitor/core@^8.4.0`
- `@capacitor/cli@^8.4.0`
- `@capacitor/android@^8.4.0`

**Files created for the Android build:**

- `capacitor.config.ts` — appId `com.haven.app`, appName `Haven`, webDir `dist/client`, splash config
- `scripts/build-capacitor.js` — emits `dist/client/index.html` from the Vite build
- `src/lib/native-fetch-bridge.ts` — rewrites relative fetches to the remote backend when running natively
- `src/router.tsx` — uses `createHashHistory()` on native (`file://`) so route navigation works without a server
- `package.json` scripts: `cap:build`, `cap:sync`, `cap:open`

The `android/` folder itself is generated by `npx cap add android` and is
**not** committed (Capacitor regenerates it from `capacitor.config.ts`).

## Manual steps you still need to do

1. **Install Android Studio + JDK 17** on your machine.
2. Run the four commands in *Step-by-step* above (`bun install`, `bun run cap:build`, `npx cap add android`, `npx cap sync android`).
3. Build the APK from Android Studio or `./gradlew assembleDebug`.
4. (Only if you want Google sign-in on the APK) add your debug SHA-1 to your Google OAuth client.

That's it — `app-debug.apk` will be ready to install on any Android device.
